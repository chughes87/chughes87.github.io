🔌 Connection Rules

GOTCHA #3: Hot vs Cold Inlets Messages trigger LEFT to RIGHT. Connect the RIGHT inlet first (cold), then LEFT inlet (hot) to avoid incorrect calculations.
GOTCHA #4: Signal vs Message Thick lines (~) = audio signals (44.1kHz+)
Thin lines = control messages (events)
Can't directly mix! Use [sig~] or [snapshot~]
Conversion: [sig~] = message → signal
[snapshot~] = signal → message

⚠️ More Gotchas

GOTCHA #5: Feedback Loops Message feedback without [delay] = CRASH. Audio feedback is OK but watch for runaway DC offset.
GOTCHA #6: Bang to Output Many objects need a bang to trigger output. Use [bng] or [bang] to trigger.
GOTCHA #7: Symbols vs Numbers "0" (symbol) ≠ 0 (number). Type matters!
GOTCHA #8: Initialization Use [loadbang] to set defaults when patch opens. Critical for filters and oscillators!
TIP: Debugging Use [print label] to see message values in the console. Essential for debugging control flow.

🎨 Farnell's Approach

Procedural Audio Design Build sounds from physical principles, not samples. Think: what makes this sound in real life?
Signal Chain Thinking Source → Modulation → Filtering → Envelope → Output. Understand each stage's purpose.
Start Simple Begin with basic waveforms. Add complexity incrementally. Test at each stage.
Use Your Ears Theory guides, but your ears decide. Tune parameters by listening, not just math.
